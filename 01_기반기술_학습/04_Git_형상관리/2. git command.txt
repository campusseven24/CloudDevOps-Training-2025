1. Git 기초 명령어
    1) git 버전 확인
        - git --version 
    2) 현재 git 설정 확인 
        - git config --list        
    3) 사용자 정보가 없다면 설정
        - git config --global user.name kim
          git config --global user.email kim@gmail.com        

2. 파일의 Life Cycle
    1) Working 디렉토리의 모든 파일은 Tracked와 
       UnTracked로 나뉨
    2) UnTracked
        - 관리 대상이 아님을 나타내며, 스냅샷에도,
          Staging Area에도 포함되지 않은 파일 
    3) Tracked 
        - 관리 대상임을 나타내며, 
        - UnModified(수정하지 않음), Modified(수정함),
          Staged(커밋으로 저장소에 기록할) 상태로 구분됨.          


3. Git의 세 가지 단계  
    1) Working 디렉토리 
    2) Staging Area
    3) Git 디렉토리 

4. Git의 세 가지 상태
    1) Modified
        - 수정한 파일을 아직 로컬 데이터베이스에 커밋하지 
          않은 상태 
    2) Staged
        - 현재 수정한 파일을 곧 커밋할 것이라는 상태 
    3) Committed
        - 데이터가 로컬 데이터베이스에 안전하게 저장된 상태                   

5. Git 저장소(repository)          
    1) Git 저장소 초기화 
        - git init
    2) .git 폴더 확인 (숨김 폴더)
        - ls -al 
    3) 저장소 상태 확인
        - git status 
    4) Staging Area에 파일 추가하기 
       Staging Area에 변경 사항 올리기 
        - git add 
       Staging Area에 여러 파일 동시에 추가 
        - git add style.css script.js 
       모든 변경사항 스테이징 
        - git add * 
    5) Commit 하기, 스테이징한 파일 커밋하기  
        - git commit 
        - git commit -m "메시지"
        - 버전 만드는 것을 깃에서는 커밋(commit)한다고 함.

        - git commit -am "메시지"
            - 스테이징과 커밋 한꺼번에 처리하기 

    6)  커밋 히스토리 확인
        - git log 
        - git log --oneline   
        - git log --oneline --graph   
        - git log --stat
            - 커밋 통계 
        - 커밋 해시(commit hash), 깃 해시((git hash)
            - 커밋을 구별하는 아이디.

    7) 변경 사항 확인하기 
        - git diff 

    8) 방금 커밋한 메시지 수정 
        - git commit --amend     

6. 작업 되돌리기 
    1) 작업 트리에서 수정한 파일 되돌리기 
        - git restore
            - 파일을 수정했을 때 스테이징하지 않은 상태에서 
              수정을 취소함 
        - git restore --staged
            - 수정된 파일을 스테이징까지 했을 때 스테이징을
              취소하는 방법               
    2) 최신 커밋 되돌리기 
        - git reset HEAD^
            - 수정된 파일을 스테이징하고 커밋까지 했을 때
              가장 마지막에 한 커밋을 취소하는 방법 
            - 파일을 unstaged 상태로 작업 디렉토리에 보관함
        - git reset --mixed HEAD^

        - git reset --soft HEAD^ 
            - 커밋을 취소하고 파일을 staged 상태로 작업 
              디렉토리에 보관함 

    3) 특정 커밋 되돌리기 
        - git reset 해시 
            - 예) git reset A 
                - A 커밋 이후에 만들었던 커밋을 삭제하고 
                  A 커밋으로 이동함  
        - git reset --hard 복사한 커밋 해시 

    4) 커밋 변경 이력 취소하기 
        - 변경 사항만 취소하고 커밋은 남겨 두는 경우 
        - git revert
