2. 패키지 
    1) 사용자와 같은 위치
        - 같은 패키지에 소속되어 있음
        - 패키지 경로를 생략해도 됨
    2) 사용자와 다른 위치
        - 서로 다른 패키지임.
        - 패키지 전체 경로를 포함해서 클래스를 적어줘야 함

3. import 
    1) 코드에서 첫줄에는 package를 사용하고, 
       다음 줄에는 import를 사용함 
        - import를 사용하면 다른 패키지에 있는 클래스를 가져와 사용 가능함
        - 특정 패키지에 포함된 모든 클래스를 포함해서 사용하고자 할때
            - import 시점에 *(별)을 사용하면 됨        

4. 클래스 이름 중복 
    1) 클래스 이름이 같아도 패키지 이름으로 구분해서 
       같은 이름의 클래스를 사용할 수 있음.

5. 패키지 규칙
    1) 패키지의 이름과 위치는 폴더(디렉토리) 위치와 같아야 함 (필수)  
    2) 패키지 이름은 모두 소문자를 사용함 (관례)
    3) 패키지 이름의 앞 부분에는 일반적으로 회사의 도메인 이름을 
       거꾸로 사용함. (관례)
        - com.sun.jarsigner    
        - com.choongang
